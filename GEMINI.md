## 3. Directory Structure (Feature-Sliced)

The project follows a **Feature-Sliced Design** hybrid:

- **`src/app/`**: Routes & Layouts.
  - `(marketing)/`: Landing pages.
  - `(saas)/`: Protected app routes (`dashboard`, `project`, `chat`).
  - `api/`: Backend route handlers.
- **`src/features/`**: Domain logic.
  - `builder/`: The core project wizard & writing workspace.
  - `bot/`: AI chat personalities (Jay, Nengi).
  - `auth/`: Authentication logic.
- **`src/services/`**: Business logic & external API wrappers (`ai.service.ts`, `paystack.service.ts`).
- **`scripts/`**: Utility scripts for ops (promoting admins, seeding data).

## 4. Critical Workflows

### Development
- **Start:** `pnpm dev` (Runs on `localhost:3000`)
- **Database:**
  - `pnpm prisma generate` (Regenerate client)
  - `pnpm prisma migrate dev` (Apply migrations)
  - `pnpm prisma studio` (View data)

### AI Generation Flow
1. **User Input:** Chat with Jay or use Builder Wizard.
2. **Processing:** Requests go to `/api/generate/*`.
3. **Execution:** `AI.service.ts` calls Vercel SDK.
4. **Storage:** Results saved to DB (PostgreSQL).

### Payments
- **Provider:** Paystack.
- **Flow:** Initialize transaction -> User pays -> Webhook verifies -> `PaymentService` updates DB.
