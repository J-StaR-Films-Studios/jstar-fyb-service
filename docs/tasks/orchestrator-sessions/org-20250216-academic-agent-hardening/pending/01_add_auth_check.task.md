## 📚 Context

### Current Issue

The chat API route at `src/app/api/projects/[id]/chat/route.ts` does not verify that the requesting user is authenticated or has access to the specified project. This allows potential unauthorized access to project data.

### Expected Behavior

1. **Unauthenticated Request** → 401 Unauthorized
2. **Authenticated but No Project Access** → 403 Forbidden
3. **Authenticated with Project Access** → Continue with request

### Step 2: Add Auth Check at Start of POST Handler

```typescript
export async function POST(req: Request, { params }: { params: Promise<{ id: string }> }) {
  // --------------------------------------------------------
  // 0. AUTHENTICATION CHECK
  // --------------------------------------------------------
  
  const session = await auth();
  
  if (!session?.user?.id) {
    return new Response(
      JSON.stringify({ error: 'Authentication required' }),
      { status: 401, headers: { 'Content-Type': 'application/json' } }
    );
  }
  
  const { id: projectId } = await params;
  
  // --------------------------------------------------------
  // 0.5. AUTHORIZATION CHECK
  // --------------------------------------------------------
  
  const project = await prisma.project.findUnique({
    where: { id: projectId },
    select: { userId: true },
  });
  
  if (!project) {
    return new Response(
      JSON.stringify({ error: 'Project not found' }),
      { status: 404, headers: { 'Content-Type': 'application/json' } }
    );
  }
  
  if (project.userId !== session.user.id) {
    return new Response(
      JSON.stringify({ error: 'Access denied' }),
      { status: 403, headers: { 'Content-Type': 'application/json' } }
    );
  }
  
  // Continue with existing logic...
}
```
